## 2024-04-27 - Buffer Overflow in PIN and Device Name Buffers
**Vulnerability:** Several static buffers (like `pin_char[6]`) were exactly the size of the logical string ("123456" for a 6-digit PIN) without leaving room for the null terminator (`\0`), leading to a classic off-by-one stack buffer overflow. Additionally, formatting an implicitly signed `char` representation of a MAC hash piece using `%02X` could result in sign extension to `FFFFFFXX` instead of `XX`, overwriting the `bt_devname` array bounds.
**Learning:** Hardcoded buffer sizes attached to formatting functions (`sprintf`) are fragile, and implicit types (like signed vs unsigned chars) can cause unintended string lengths when cast to formatted string outputs like hex sequences.
**Prevention:** Use `snprintf` everywhere to ensure buffer boundaries are strictly respected, correctly size arrays for strings with null-terminator bytes, and explicitly cast `char` fields to `uint8_t` when writing raw byte outputs as formatted hex characters.
