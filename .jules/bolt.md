## $(date +%Y-%m-%d) - [Optimize SPI Block Transfers for SX126x/SX128x]
**Learning:** Replacing byte-by-byte `SPI.transfer()` loops with block transfers `SPI.transfer(buffer, size)` in high-frequency radio driver operations significantly reduces hardware setup overhead, allowing utilization of DMA and hardware FIFOs where available. For non-destructive writes, breaking the payload into 32-byte chunks using a stack buffer ensures safe memory handling without large allocations. For reads, pre-filling the target buffer via `memset(buffer, 0, size)` handles the SPI requirement of sending zeros during receive operations seamlessly.
**Action:** When working with low-level SPI communication in C++, systematically prefer `SPI.transfer(buffer, size)` for multi-byte operations and utilize safe chunking for immutable inputs.
