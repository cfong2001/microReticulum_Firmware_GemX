
## $(date +%Y-%m-%d) - LoRa SPI Block Transfer Optimization
**Learning:** In Arduino LoRa drivers (SX126x/SX128x), replacing byte-by-byte `SPI.transfer` loops with block transfers `SPI.transfer(buffer, size)` significantly improves performance by enabling DMA or hardware FIFOs. However, `SPI.transfer` mutates the input buffer in-place with MISO data. For non-destructive writes of `const` buffers, data must be copied to a local buffer. To prevent stack overflows on memory-constrained AVR devices while avoiding heap allocation (which causes fragmentation), you must copy data in chunks to a small fixed-size stack buffer (e.g., `uint8_t temp[32]`). For block reads, the buffer must be pre-filled with `0x00` using `memset` before transfer.
**Action:** Always use chunked stack buffers for block SPI writes of `const` data. Always pre-fill buffers with `memset` before block SPI reads. Include `string.h` for `memcpy` and `memset` and always wrap these memory operations in `if (size > 0)` checks to prevent undefined behavior with potential null pointers. Always add explanatory comments directly in the code when implementing performance optimizations as per the core directives.
