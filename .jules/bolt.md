## 2024-05-15 - SPI Block Transfers in LoRa Drivers
**Learning:** Found a specific codebase optimization pattern where replacing byte-by-byte `SPI.transfer` loops with bulk `SPI.transfer(buffer, size)` significantly improves efficiency on LoRa drivers (SX126x/SX128x), as it reduces loop overhead and makes block DMA possible. However, `SPI.transfer` mutates the input buffer, so writes require copying to a fixed-size stack buffer (`uint8_t temp[256]`) first, and reads require pre-filling with zeros using `memset`.
**Action:** When working on hardware driver components, proactively look for opportunities to replace byte loops over SPI/I2C with block data transfer functions, ensuring proper input state preservation and safety checks like `size > 0` before `memcpy`/`memset` to avoid undefined behavior.
